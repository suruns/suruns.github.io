HashCode

java中object类有hashcode方法，所有自定义类都可以覆写此方法（一般是跟equals一起），今天突然明白了为什么hashcode不能写成定值，例如:


原因如下：
背景：Java中有一种集合叫hashset,hashset在添加元素的时候会先检测目标元素的hashcode值，若hashcode有相同的，则执行equals方法，若equals返回true,则认为有重复数据，不再添加。
但是如果equals返回false呢？则将目标元素挂在同一个hashcode下，如下图:

如图所示，相同的hashcode但是equals返回false的元素被放在同一个哈希桶里面，而hashcode就相当于字典的目录一样，在我们获取元素时可以通过hash算法快速查找元素，
但是：如果我们偷懒将所有的hashcode方法都返回了同一个定值，那么他们放入hash集合时，所有的元素都会在同一个hash桶里面，我们查找元素时，就相当于遍历查找，那么hash算法也就失去作用了.
